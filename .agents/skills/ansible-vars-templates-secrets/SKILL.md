# Ansible Vars Templates Secrets

## Purpose

Change Ansible configuration safely by separating secrets, shared vars, role defaults, and rendered templates.

Use this when:

- Adding an environment variable to a deployed service.
- Moving hardcoded values into vars.
- Adding a secret-backed config value.
- Keeping app settings and deployment templates aligned.

## First Questions

- Is the value secret?
- Is it global/shared or role-local?
- Which template consumes it?
- Which service must restart when it changes?
- Does application code need a matching setting?
- Does local/dev config need a default?

## Placement Rules

- Secrets: encrypted vault files, commonly `inventory/group_vars/*/vault.yml`.
- Shared non-secrets: inventory/group vars.
- Role-specific defaults: `roles/<role>/defaults/main.yml`.
- Rendered config: `roles/<role>/templates/*.j2`.
- Static non-secret files: `files/`.

Never put plaintext secrets in defaults, normal vars, templates, comments, README snippets, or tests.

## Naming

Use prefixes that reveal ownership:

- `api_*` for API service settings.
- `worker_*` for worker settings.
- `db_*` for database settings.
- `vault_*` for secrets.

Avoid duplicating the same value under multiple names. Prefer one canonical var consumed by multiple templates.

## Template Pattern

```jinja
# Generated by Ansible. DO NOT EDIT MANUALLY.

APP_ENV={{ app_environment }}
PUBLIC_URL={{ app_public_url }}
DATABASE_URL={{ vault_database_url }}
```

Keep templates readable. Prefer a template over many `lineinfile` tasks.

## Change Checklist

For a new config value:

1. Add app setting/default in code if needed.
2. Add non-secret var or vault secret.
3. Render it in the env/config template.
4. Notify the service restart handler from the template task.
5. Update tests or syntax checks.
6. Document only if the project already has operator docs for config.

For cross-role values such as domains, ports, paths, and URLs:

- Put them in shared vars.
- Reference them from role defaults/templates.
- Check every consuming service.
- Keep public URLs, internal URLs, proxy upstreams, and app settings consistent.

## Validation

Use:

```bash
ansible-playbook --syntax-check playbooks/site.yml
ansible-lint
```

Do not decrypt, print, or commit vault secret values.
